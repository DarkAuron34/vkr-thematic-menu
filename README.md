# Веб-система для составления тематических меню по многокритериальному запросу

![Django](https://img.shields.io/badge/Django-4.0.4-092E20?style=flat&logo=django)
![Python](https://img.shields.io/badge/Python-3.10-3776AB?style=flat&logo=python)
![SQLite](https://img.shields.io/badge/SQLite-003B57?style=flat&logo=sqlite)

Этот проект представляет собой веб-приложение, разработанное для автоматизации процесса составления тематических меню (например, для праздников, свадеб, дней рождения) и поиска подходящих ресторанов на основе сложного многокритериального запроса пользователя.

Проект разработан в рамках Выпускной квалификационной работы (ВКР) бакалавра в Волжском политехническом институте (филиал ВолгГТУ) в 2022 году.

## 📌 Основные возможности

### 🍽️ Для поиска и составления меню:
* **Многокритериальный поиск рецептов:** фильтрация по названию, стоимости, времени приготовления, сложности, категории блюд и национальной кухне.
* **Умный фильтр ингредиентов:** возможность как включать обязательные ингредиенты, так и исключать нежелательные (например, аллергены).
* **Автоматическая генерация меню:** система сама подбирает список блюд на основе заданных параметров и тематики события.
* **Кастомизация сгенерированного меню:** возможность точечно заменить любое блюдо в предложенном меню на другое, подходящее по тем же критериям.

### 🏢 Для поиска ресторанов:
* Поиск заведений по городу, тематике (для проведения конкретных мероприятий) и национальной кухне.

### 👤 Пользовательские функции:
* Регистрация и авторизация пользователей.
* Личный кабинет с сохранением истории активного меню.
* Система избранного: возможность добавлять в "Избранное" понравившиеся рецепты, готовые меню и рестораны.
* Создание собственных рецептов и добавление их в общую базу данных.

## 🛠 Технологический стек
* **Backend:** Python, Django (Архитектура MVT)
* **База данных:** SQLite (встроенная) / Поддержка ORM Django
* **Frontend:** HTML5, CSS3, JavaScript (шаблонизатор Django)
* **Тестирование:** CogTool (оценка интерфейса), Webserver Stress Tool 8 (нагрузочное тестирование)

## 📂 Структура базы данных
Система включает в себя продуманную реляционную структуру данных (модели):
* `Dish`, `Ingredient`, `DishCategory`, `Difficulty`, `NationalCuisine`, `Theme` — для описания блюд и их свойств.
* `Menu` — для сохранения сгенерированных наборов с привязкой к JSON-параметрам поиска.
* `Restaurant`, `City` — для поиска заведений.
* `Profile` — расширение стандартной модели `User` для реализации функционала "Избранного".
